# brand_voice.md 模板 + 重跑保留規則

## 重跑時務必保留使用者手動修正（強制）

寫入前先檢查 `brand_voice.md` 是否已存在：

1. 不存在 -> 直接用下方模板產生新檔。
2. 已存在 -> **先讀一遍，保留所有使用者修改過的內容**：
   - 逐字保留 `## 使用者手動修正` 整段，不重新生成、不摘要。
   - 掃過其他每一段，找出看起來是使用者自己寫的內容（如 `> user:` 開頭的行、跟原本生成分析矛盾的修改、生成器不會自己寫出來的新條目）。有疑慮就保留，不要蓋掉。
   - 覆寫前先給使用者看一段簡短 diff 摘要：「我要重新生成 N 個段落，你的手動修正段落和以下編輯會保留：[...]。確認嗎？」
   - 確認後合併：自動段落重新生成，使用者編輯過的內容原樣保留，手動修正段落整段照抄。
   - 合併有疑義時，**停下來問使用者**，不要自己決定覆寫。

`## 使用者手動修正` 段落只要非空就絕不覆寫，沒有例外。

## 完成回報（Step 4）

分析完後，依這個順序回報使用者：

1. 分析了幾篇貼文、資料量等級（見 SKILL.md 的等級表）。
2. 挑 2–3 個最明顯的風格特徵，各附一段簡短原文範例。
3. **明確說明這是第一版草稿，不是最終版**：
   > 「我根據你的貼文生成了第一版 Brand Voice 草稿。旁觀者永遠會漏掉一些你自己才知道的東西——麻煩你讀過一遍，把我判斷錯的地方改掉，並補上『使用者手動修正』段落。你的修改才是這份檔案真正有用的地方。」
4. 指出哪些段落最需要使用者補充（禁忌用語、「這不是我」的例子、讀者分群稱呼）。
5. 誠實列出哪些維度資料太薄、分析比較粗略。
6. 邀請使用者回饋：「有哪裡覺得不對，跟我說，我直接改檔案。」

不要說「你的 Brand Voice 已經完成」，一定要帶上「這是草稿」的但書。
