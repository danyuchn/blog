## 1. 溝通技巧（最重要）

### ✅ 清晰描述需求（148 個案例強調）

**核心原則：**
- 用**自然語言**表達，不需要技術術語
- 提供**背景資訊**和**目標**
- 說明**為什麼**要做，不只是**做什麼**

**案例範例：**

❌ **不好的描述：**
```
幫我處理一下這些檔案
```

✅ **好的描述：**
```
我有 100 個 Markdown 檔案在 /content 資料夾，
每個檔案的標題目前是英文，我想把它們都改成中文，
因為我的部落格受眾是台灣讀者。
請幫我批次處理，並保留原始檔案的備份。
```

---

### ✅ 提供足夠的上下文

**關鍵資訊包括：**
- 檔案路徑或 URL
- 預期結果
- 限制條件（時間、成本、技術限制）
- 失敗的錯誤訊息（如果有）

**案例範例：**
```
我的網站 https://example.com 突然打不開。
錯誤訊息顯示：「502 Bad Gateway」。
網站是用 WordPress 架設，託管在 AWS。
昨天我更新了一個外掛，之後就出現這個問題。
請幫我診斷並修復。
```

---

### ✅ 分段描述複雜需求

**多步驟任務的描述方式：**

1. 每天早上 8 點抓取 AI 相關新聞（來源：TechCrunch, VentureBeat）

## 2. 任務規劃與分解

### ✅ AI 如何規劃任務（37 個案例）

**AI 的典型規劃流程：**
1. **理解背景**：讀取相關檔案、探索專案結構
2. **拆解任務**：將大任務分解為小步驟
3. **執行驗證**：每個步驟完成後檢查結果
4. **調整策略**：遇到問題時尋找替代方案

**你可以這樣引導 AI：**
```
在開始之前，請先：
1. 探索現有的專案結構
2. 列出你打算執行的步驟
3. 告訴我每個步驟可能遇到的風險
```

## 3. 工具使用與理解

### ✅ Claude Code 常用工具

AI 會在背後使用這些工具，你不需要懂技術細節，但理解它們的用途有助於協作：

| 工具 | 用途 | 使用者視角 |
|------|------|-----------|
| **Read** | 讀取檔案內容 | AI 需要了解現有程式碼或文件 |
| **Write** | 建立新檔案 | AI 產出結果（腳本、報告、設定檔） |
| **Edit** | 修改現有檔案 | AI 更新程式碼或文件 |
| **Bash** | 執行終端指令 | AI 執行命令（git、npm、測試） |
| **Grep** | 搜尋檔案內容 | AI 查找特定程式碼或文字 |
| **Glob** | 搜尋檔案名稱 | AI 尋找符合模式的檔案 |
| **WebSearch** | 網路搜尋 | AI 查詢最新資訊或解決方案 |
| **WebFetch** | 抓取網頁內容 | AI 讀取線上文件或資料 |
| **Task** | 委派子任務 | AI 啟動專門的 agent 處理複雜工作 |

## 4. 錯誤處理策略

### ✅ AI 遇到問題時的行為模式

**AI 會：**
1. **解釋原因**：清楚說明為什麼失敗
2. **提供替代方案**：建議其他做法
3. **請求協助**：詢問你需要什麼資訊或權限

**案例：無法存取私人網頁**
```

    你可以：

    2. 截圖給我看，我可以手動輸入數據
    3. 使用銀行提供的 API（如果有）
```

## 5. 協作流程優化

### ✅ 迭代式協作（不是一次完成）

**理解協作的本質：**
- AI 不是魔法，無法一次完美完成
- 需要**多輪對話**來調整和優化
- 你的反饋是改進的關鍵

**典型迭代流程：**
```
第 1 輪：AI 提供初步方案
第 2 輪：你提出調整需求
第 3 輪：AI 修正並執行
第 4 輪：驗證結果，發現小問題
第 5 輪：AI 微調，完成任務
```

---

### ✅ 主動確認與驗證

**在關鍵步驟要求確認：**
```
在你執行之前，請先：
1. 列出會被修改的檔案
2. 說明可能的影響
3. 確認有備份或版本控制
```

**執行後的驗證：**
```
執行完成後，請幫我：
1. 檢查結果是否正確
2. 執行相關測試
3. 確認沒有破壞現有功能
```

## 7. 常見陷阱與解決方案

### ❌ 陷阱 1：描述過於模糊

**問題：**
```
幫我優化網站
```

**AI 無法知道：**
- 優化什麼？速度、SEO、UI？
- 目標是什麼？
- 有什麼限制？

**改進：**
```
我的網站載入速度很慢（首屏時間 > 5 秒）。
請分析效能瓶頸，並提出優化方案。
目標是將首屏時間降到 2 秒以內。
```

---

### ❌ 陷阱 2：期待 AI 讀心術

**問題：**
```
做得不對，重做
```

**AI 不知道哪裡不對。**

**改進：**
```
這個結果有問題：
1. 檔名應該是日期格式（YYYY-MM-DD），而不是數字
2. 內容應該是繁體中文，但你給了簡體
3. 缺少了「摘要」欄位

請按照這 3 點修正。
```

---

### ❌ 陷阱 3：不提供錯誤訊息

**問題：**
```
執行失敗了，怎麼辦？
```

**AI 需要看錯誤訊息才能診斷。**

**改進：**
```
執行失敗了，錯誤訊息如下：
[錯誤訊息完整內容]

我是在執行「npm install」時出現這個錯誤。
之前沒有這個問題，今天才開始發生。
```

---

### ❌ 陷阱 4：跳過驗證步驟

**問題：**
直接要求 AI 執行關鍵操作（刪除、部署、發送郵件）

**風險：**
- 誤刪重要資料
- 部署錯誤版本
- 發送測試郵件到正式客戶

**改進：**
```
在刪除這些檔案之前，請：
1. 列出會被刪除的檔案清單
2. 確認有備份
3. 等我確認後再執行
```

---

### ❌ 陷阱 5：忽略 AI 的建議

**問題：**
```
AI：這個操作有風險，建議先備份
使用者：不用，直接執行
[結果：資料遺失]
```

**改進：**
聽取 AI 的風險提示，至少花 30 秒評估。
