這部影片我想做很久了，但是卻一直沒做，直到今天。

因為我覺得 Obsidian 對我來說是個很特別的存在。他的自由度太高，所以在每個人的手上會長成不同的樣子。但是他的成效又不是立刻就能看到的，而是越長越大你才會發現離不開它。

所以這部影片要怎麼拍我想了很久。最後我決定回歸最自然的方法——

跟你們誠實分享我是因為什麼才接觸 Obsidian、我的知識庫長什麼樣子、它是如何慢慢長大的、以及你要怎麼開始你的第一步，用很簡單的指令建構一個適合你自己的、初始的知識庫。

## 為什麼我接觸 Obsidian

我會用 Obsidian 不是因為它有多漂亮的 UI，也不是因為什麼生產力 YouTuber 推薦。是因為我用 Claude Code 之後，發現一件事：

**模型的長期記憶不能存在模型那裡，要存在我這裡。**

我每天跟 Claude Code 講的東西、做的決定、踩的坑、學到的教訓——這些東西如果只存在當下的對話 context 裡，下個月再開新對話的時候，模型完全不知道我之前發生過什麼。我變成每次都要從頭講起。

Obsidian 的解法是：我把所有東西寫成 markdown，存在本機，用 wikilink 互連。Claude Code 在每次 session 啟動時自動讀我指定的入口檔案（譬如 `MEMORY.md`）。這樣模型就有了「跨 session 持久化記憶」——但這個記憶的所有權在我手上，不在 Anthropic 那邊。

## 我的知識庫長什麼樣子

幾個主要的資料夾：

- `daily/YYYY-MM-DD.md`：每日筆記。完成事項、決定、踩坑、教訓
- `weekly/YYYY-MM-DD.md`：每週覆盤
- `projects/`：每個進行中的專案一個檔案，記載當前狀態、待辦、決策歷史
- `pillars/`：人生的幾個主軸（事業、健康、關係、財務），每個一個檔案
- `bookmarks/`：讀到的好東西、值得記下來的工具、書摘
- `decisions/`：重要決策的脈絡記錄（不只結論，記下「為什麼這樣選」）
- `personal/`：私密區，跟療癒、伴侶、家人有關
- `archive/`：已完成的舊任務歸檔

兩條動脈：

- **時間軸**（daily / weekly）：什麼時候發生什麼事
- **主題軸**（projects / pillars / decisions）：每個主題的最新狀態

互查：每天的 daily 用 wikilink 連到當天觸碰的主題檔；主題檔的「Related」段落反向連回相關 daily。這樣不管從哪邊進去，都能找到完整脈絡。

## 它如何慢慢長大

我用 Obsidian 大概一年半。一開始就只有 daily 跟 projects 兩個資料夾。當時也沒有刻意設計結構——就每天寫，遇到新類型才開新資料夾。

幾個轉折點：

- **3 個月時**：發現很多東西重複寫在不同 daily 裡，開了 `pillars/` 把長期主題抽出來
- **6 個月時**：開始用 Claude Code 自動讀 `MEMORY.md`，知識庫進入「AI 可讀」階段
- **9 個月時**：跨檔 wikilink 數量爆炸（單檔 outbound > 50），開了 `weekly/` 做匯總層
- **12 個月時**：開始做「主檔同步掃描」（每週檢查主檔狀態行 vs section header 是否一致），抓出兩三個悶了 3 週的矛盾

這個成長曲線是**滾出來的，不是規劃出來的**。如果你一開始就想設計一個完美結構，你會卡在「該怎麼分類」上面三個月不寫東西。

## 你的第一步

不要從複雜結構開始。第一週就做兩件事：

1. 建一個 `daily/` 資料夾，每天寫一個 `YYYY-MM-DD.md`
2. 寫的時候有任何專有名詞（人名、專案名、工具名）就用 `[[名字]]` 包起來

不需要刻意創建那個 wikilink 對應的檔案。Obsidian 會自動把 `[[ABC]]` 標記成「未存在的連結」。當你某天覺得需要把 ABC 抽出來成獨立主題，再點那個連結建立檔案就好。

這個工作流的核心是「**寫完才整理，不要先設計再寫**」。寫了兩個月之後你會自己看見哪裡需要整理，哪些主題該抽出來，哪些重複可以合併。

## 在前陣子 markdown 揚棄論的風頭中

前陣子有一波討論說「該揚棄 markdown，HTML 才是未來」。我這個時間點發 Obsidian 影片，算不算是一種逆風？

我不覺得這是逆風或順風的問題。Markdown 跟 HTML 在不同場景各有適合的位置。Markdown 適合「給未來的我 + AI 一起看」，HTML 適合「給其他人看」。前者是知識庫的場景，後者是發布的場景。

把兩個拿來互比，就像問「該買運動鞋還是皮鞋」一樣——看你下一步要去打球還是去開會。

我接下來還會繼續用 Obsidian。也歡迎你從第一個 daily note 開始。
