Stephen Covey 在《高效能人士的七個習慣》（與成功有約）裡提出的時間管理矩陣，是兩軸切四象限：**重要 × 緊急**。

| | **緊急** | **不緊急** |
|---|---|---|
| **重要** | Q1 危機處理 | Q2 自我提升 |
| **不重要** | Q3 被打斷 | Q4 浪費 |

Covey 的核心主張是：

1. **要事第一，不是急事第一**
2. **多數人活在 Q1+Q3**，被「緊急」綁架，誤把緊急當重要
3. **高效能者刻意擴大 Q2**（重要但不緊急）：規劃、學習、健康、深度工作、關係——是 Q2 沒做好才會產生 Q1 危機
4. **Q3 是最大陷阱**：別人的急事不是你的要事，學會說不
5. **Q4 砍掉**

我自己用這個框架用了很多年。最近開始想：把 Claude Code 放進這四格之後，會發生什麼事？

## 矩陣 × Claude Code 協作

| | **緊急** | **不緊急** |
|---|---|---|
| **重要** | **Q1 危機處理**<br/>讓 Claude Code 加速：迅速 fetch 資訊、迅速產出，把處理時間壓縮 | **Q2 自我提升**<br/>因為 Q1/Q3 被 AI 吃掉，多出來的時間用來與 AI 深度討論、規劃、把事情真的落地 |
| **不重要** | **Q3 被打斷**<br/>由 Claude Code 代為處理（草稿信、查資料、行政流程），減少自己浪費在這的時間 | **Q4 浪費**<br/>因為前三格已被 AI 加速，剩下這格自然萎縮 |

## 四格分別發生什麼事

**Q1：加速**

危機本身沒有變少，但每一個危機的處理時間被壓縮。客戶今天傳來一份要 3 小時讀完才能回覆的合約，Claude Code 30 分鐘就能讀完、標出風險條款、產出三版回覆草稿。我只需要在最後挑一版、改幾句，按下傳送。

**Q2：擴大**

這是真正的關鍵。AI 協作的長期複利不在 Q1，而在 Q2。

Q1 跟 Q3 被 AI 吃掉之後，你會多出每天 2-3 小時。多數人這時候會掉進兩個陷阱之一：

- 用多出來的時間做更多 Q1（接更多案子、回更多信）
- 用多出來的時間滑手機（Q4 反彈）

正確的用法是**把多出來的時間搬到 Q2**：跟 AI 一起做策略規劃、寫長文反思、學一個新領域、把你之前一直想做但沒時間做的事情真的開始做。AI 在 Q2 不是工具，是討論對象。你需要的是「一個能跟你來回辯論策略、不會累、不會評判你的人」。

**Q3：代理**

這格是很多人意外的入口。Q3 是「別人的急事」——同事的草稿、客戶的查詢、廠商的報價、行政流程。這些事重要性低，但會把你的注意力切碎。

把 Q3 交給 Claude Code 代理：
- 草稿信件先讓 AI 寫一版，你只挑一版改幾個字
- 查資料的工作直接 Claude Code WebSearch + 整理
- 行政流程（填表、上傳、申請）寫成 skill，自動跑

關鍵心態：**不要再說「這個我自己 5 分鐘就做完了」**。5 分鐘 × 一天 20 次 = 100 分鐘。這 100 分鐘搬到 Q2 才是價值所在。

**Q4：自然萎縮**

前三格都被 AI 加速之後，Q4 不需要刻意砍。你會發現自己在滑短影音的時候，心裡會浮現「我這時候可以拿來跟 AI 做策略討論」——不是因為自律，是因為機會成本變高了。

## 課程開場的應用

我在 AgentCrew Academy L1/L2 課程裡，把這個矩陣當「為什麼學 Claude Code」的開場框架。學生最有感的是 Q1 加速 + Q3 代理（因為立即可見），但 Covey 的真正洞察是要把節省下來的時間搬到 Q2，這才是 AI 協作的長期複利。

如果你也想用這個框架介紹 AI 工具給朋友或同事，建議的話術是：

> AI 不是讓你做更多事，是讓你終於有時間做你一直想做但沒空做的那件事。

這句話打中的是 Q2，不是 Q1。
