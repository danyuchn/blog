普通的週回顧，你會幫自己找理由。

「這件事沒做，是因為那件事卡住了。」「下週一定補上。」「整體還算OK。」

Mirror System 設計來防止這件事。

## 什麼是 Mirror System

方法來自 Tom's Guide 的一篇文章，核心很簡單：把你這週的 daily notes 全部餵給 Claude，然後叫它做一件事——找出你可能不願意正視的模式。

不要肯定句開頭。不要安慰。用事實語氣。

Claude 不知道你累了，不知道你有理由，不知道你希望聽到什麼。這正是它有用的地方。

## 怎麼跑

每週固定例行事項做完後，繼續在同一個 session 執行：

1. 請 Claude 讀取本週所有 `daily/YYYY-MM-DD.md`（週一到今天）
2. 貼上下面的 Mirror Prompt

## Mirror Prompt

```
以下是我這週的 daily notes。請幫我做 Mirror 分析，找出我自己可能不願意正視的模式：

1. 反覆推遲的項目
   不要問為什麼，直接指出哪些事情被反覆改期或延後，說明推遲了幾次。

2. 說要做但沒做的事
   對比 daily notes 中出現的承諾、「待辦」、「下一步」，找出整週沒有後續記錄的項目。

3. 時間花費 vs 北極星目標的落差
   分析各類工作的時間佔比，指出哪類工作「看起來有生產力但對北極星貢獻低」。

4. 整週零進展的領域
   哪個核心方向完全沒有出現，或只有維護性動作？

請用事實語氣，不要安慰，不要肯定句開頭。
```

第 3 點的「北極星」那段，換成你自己的目標。

## 輸出怎麼處理

**只記錄最不舒服的那一條。** 其他洞察當下消化就好，不用強制歸檔。

原因是：如果每週把所有結論都存起來，很快就變成一個沒人看的清單。最不舒服的那條才是真正有行動壓力的訊號。

如果某一條在兩週以上持續出現，再考慮建立對應的待辦或改變工作流。重複出現才代表是結構問題，不是偶發。

## 為什麼用 Claude 而不是自己反省

不是因為 Claude 比你聰明，是因為它不在乎你的感受。

你在自我回顧時，大腦會自動保護你：選擇性記憶、合理化、把「沒做」解釋成「暫緩」。Claude 沒有這個機制。

它看到的只有你寫下來的文字，然後告訴你那些文字裡的模式是什麼。

Mirror 的價值不在讓你做更多事。在於讓你看到你一直在迴避什麼。
