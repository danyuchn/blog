我是那種 n8n 總是學不好、Claude Code 陣痛一小時後馬上上手、最終用它處理 95% 工作的人。

對我來說，n8n 堆積木，不如直接把工作流講出來。講得不清楚也沒關係，Claude 會問我問題。

## n8n 才是過渡產品

事實上我覺得 Claude Code 根本可以直接取代 n8n。n8n 的所有事情用自然語言產出腳本都能做到，沒有必要積木疊疊樂。Claude Code 加 Notion 就好。

n8n 的價值是把自動化流程視覺化、讓非技術人員也能搭建。但當 AI agent 可以直接聽你講話就執行任務的時候，那層視覺化中間層就變成多餘的了。

## 裝了一堆工具但用不到

很多人裝了 Lobster 但實際上根本沒有那麼多需求，就是為裝而裝。其實根本不用養蝦子，Claude Code 也完全可以做到全自動化。關鍵在主理人的腦子夠不夠清楚。

fomofly 能做的？Claude Code 基本上掛一個 skill「youtube-clipper」就能完成，其他的更不用講。

與其訂十幾個 AI 工具，不如訂一個 Claude Max。

## Augment 是 Claude 的免費廣告

我是從 Augment 跳到 Claude Code 的。謝謝 Augment 的免費 Claude Opus 跟免費 Gemini Pro 並列，才讓我知道原來模型也是有分聽得懂跟聽不懂人話的。

所以 Augment 的出現，其實是幫 Claude 打了免費廣告。

## 從溫室走出來就回不去了

Claude Code 唯一推薦。初期陣痛，過了之後狠甩其他 AI 工具一萬條街。

很難找到從 Code 走回 Cowork 的人。只有從溫室走出來的，沒有聽過走回溫室的。

這不是信仰，是經歷過的人都知道的事實。當你習慣了直接跟 AI 對話、讓它自己決定用什麼工具、自己拆解任務、自己驗證結果，你就回不去那種一次只能做一件事的介面了。
