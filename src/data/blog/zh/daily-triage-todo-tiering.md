一個用 Claude Code 使用習慣的小發現：待辦分級制。

我都是用它來處理非編程事務，過去我的習慣是把待辦全部列出來，然後按照順序一個一個做，雖然會開不同 repo 的 session，但本質上還是在每個 repo 中單點推進，如果有哪個東西要我跟他討論或者審核就要卡很久。導致我每天工作的時間還是偏長，身體也因此出了狀況。

昨天剛好遇到 5 小時額度卡滿，等待的時間沒事可做，我就在把我的待辦先分門別類，在想哪些屬於哪個 repo、哪些是它可以自動做／需要給我審核／需要跟我積極協助／需要給我在外面手動操作，打算額度一滿就送給他。全自動的都在同一個 session，給我審核跟要跟我討論的各自一個 session，需要手動操作的我另外去做掉。

結果發現，原本會拖延一整天的待辦，竟然 1 小時內就完成了？

所以我就另外做了一個 skill。他會幫我把待辦分 repo，分完 repo 再分成 自動／審核／協作／手動 四級。只列出當日、逾期、未來七天中非阻塞、非等待他人、非等待時機的待辦事項。

設計的時候踩了一個坑。初版我讓腳本用關鍵字猜分類，但「跟進」含主動語意卻撞「等」字、「追蹤」看似自動卻需要數據累積——單字匹配一定誤判。所以腳本退回純事實層（只收集待辦、抓 context），判斷層交給 LLM 讀 text 加 context 逐條定。

以後每天早上 run 這個 skill，開大量 session 開工，驗證完就收工。減少用電腦的時間，早點睡覺，好好顧自己身體吧。
