相信我，Claude 為你準備好了一百種方式讓你花式燒 token，他也準備好了一百種藉口，把燒 token 的責任推給使用者。

## 花式燒 token 的一百種方式

你可以永遠用同一個 session，從不主動壓縮。每次訊息之間隔一小時以上。用印地語、韓語跟他對話。請他盡可能開 subagent，越多越好。所有文檔都不要寫 md，一律寫 html。

這些招數加起來，token 消耗很可觀。但官方永遠有話說：是你用法的問題，不是我們的問題。

## 那些炫富語錄

Anthropic 有兩位蠻有名的人，一位我都叫他「光頭」，一位是 Thariq。這兩位講過的話，我聽起來根本每一句都在炫富。

「我只用 Opus。」「我每次都十幾個 session 在跑。」「每個 session 下面同時掛好多 subagent。」另一位則是：「md 拜拜、html 讚讚，html 不會比較耗 token。」「限額只影響不到 10% 的人。」「雙倍用量不用想太多，禮物就是單純的禮物。」

之前原始碼洩漏，被發現 Anthropic 內部員工帳號有特殊旗標。原來人家是特權階層，難怪講話那麼輕鬆。

## 但我還是天天讀他們的技術部落格

講了這麼多酸話，我得老實說：我對 Anthropic 是又愛又恨。

恨的是那股噁心的精英心態，還有那種自以為在教育用戶、其實只是黑箱的騷操作。愛的是他們分享的技術文章——在各家模型商巨頭裡面，他們的技術部落格品質數一數二高，而且雅俗共賞，外行看得懂、內行看得到門道。

我尤其愛看資安、權限相關的那幾篇。

## 為什麼是資安

這要從我還在達克效應頂端的時候講起。那陣子我開心地 vibe coding，覺得自己無所不能。身旁有個做資安的朋友，常常給我一些珍貴的提醒，讓我慢慢意識到一件事：風險跟功能、便捷程度是呈正相關的。越方便、功能越開放，洞就越大。

後來我自己不小心把密鑰洩漏出去，吃了一筆損失，才真正開始認真複習那些當初被提醒過的教訓，開始把資安跟權限當一回事。

所以現在我常讀的東西大概是這幾類：Claude 官方部落格、Claude Code 官方文檔、每次模型發布的系統卡、還有 Thariq 在 X 上的文章。

## 一個迷人的夾心位置

讀久了會發現這是一個很迷人的世界。

Anthropic 卡在兩股力量中間。一邊是有一定機率不受控、又難以詮釋的模型；另一邊是審核疲乏、會偷懶的人類。他們要用純理性的邏輯，去設計一整套 harness 來管控模型的行為，同時又要對抗人類的怠惰——而這些人類很多是新加入、沒受過什麼專業訓練的使用者。

這種兩面夾擊的設計取捨，在他們一篇篇技術文章裡都寫得很清楚。你能看到他們怎麼權衡、怎麼妥協、怎麼在安全跟好用之間拉扯。

精英心態的部分我繼續嫌，但技術部落格我是真的推薦大家多讀。開卷有益。
