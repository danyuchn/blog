最近有機會幫一些以前的同事、團隊成員升級他們的工作流程，跟 AI 接軌。於是開始有意識地紀錄自己用 Claude Code 工作的習慣。

我很討厭那種網路上懂得皮毛就拿來賣課收割的「AI 專家」。對我來說，既然自己知道自己懂的只是皮毛，那免費分享才是最大的樂趣。

今天想分享的是：**如何下出清楚的起始指令**。

![Claude Code 終端截圖：起始指令範例](/blog/images/claude-code-initial-prompt-tips/terminal-prompt.png)

## 心法一：先想清楚最終目標和動機

你必須先想清楚你的最終目標，跟動機是什麼。

以我的例子來說，最終目標是「重構清理 CLAUDE.md 跟記憶檔」，因為我感覺距離上次清理已經有一段時間了，跟他協作的過程似乎又有一點不太協調了。

很多人一打開 Claude Code 就隨便丟一句話。但如果你自己都不清楚要什麼，AI 怎麼可能幫你做對？先花 30 秒想清楚：我到底要達成什麼？為什麼要做這件事？

## 心法二：想像沒有 AI 時你會怎麼手動操作

想像「如果沒有 AI，你本人手動會怎麼操作？流程是什麼？」

這裡我就想像：我會先去找「好的記憶檔跟規則檔該怎麼編」，然後找出我電腦裡所有的規則跟記憶檔來對照是否符合官方推薦實踐。我就用這種方式編寫指令。

這個思維方式很關鍵。你本來就會做的事情，只是要讓 AI 代勞。把你的操作流程翻譯成指令，比憑空想像一個 prompt 有效得多。

## 心法三：不確定就用計畫模式

如果你怕有什麼自己想得不周詳或忘記細節的地方，請先開計畫模式。計畫模式會主動在不清楚的地方啟動選擇題詢問你。

計畫模式是 Claude Code 裡我最推薦新手使用的功能。它會先分析現狀、擬定方案，然後在不確定的地方主動問你。你不需要一開始就把所有細節想好——讓 AI 幫你補齊。

![計畫模式分析畫面：自動診斷各檔案狀態並提出改善建議](/blog/images/claude-code-initial-prompt-tips/plan-overview.jpg)

## 心法四：不用怕打錯字

不用怕打錯字，除非你的錯字是造成語意完全偏離，不然就放他過吧。怕自己打字乏善可陳難以組織語言，用語音輸入也可以，儘量講，不用怕嗯嗯呃呃語助詞，佔不了你幾個 token。

Claude Code 的語言理解能力很強，錯字、口語、語音輸入的贅字對它來說都不是問題。不要因為追求「完美 prompt」而卡住，先講了再說。

## 心法五：切一個適合的模型

通常我會建議新模型剛出的時候看一下官方的系統卡跟介紹、看一下 Reddit 上面的網友使用心得，最重要的是，自己親身給他用個兩三天，你自然會知道不同等級的模型你願意讓他做什麼任務。

以這次的例子來說，因為不是代碼的複雜重構等高難度任務，所以不用 Opus 4.6，改而用在維護文檔任務上特別擅長的 Sonnet 4.6。模型選對，省時省錢，效果更好。

## 心法六：好好看計畫，不懂就問、不滿意就改

好好看他寫的計畫。如果你看不懂，可以在選項 4 跟他講「我看不懂，寫更白話文一點」。我的習慣是只要計畫一有不懂就問，一有不滿意就叫他改。最後直接 bypass permissions 讓他跑通，我去做他做不了的打掃家裡。

![計畫模式執行畫面](/blog/images/claude-code-initial-prompt-tips/plan-execution.jpg)

但是記得做好版本控制跟備份預防意外。如果你不信任他，那逐動作審核模式也很好。
