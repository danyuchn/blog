還記得上週看了 Claude Code 光頭分享的「AI 導入的不同階段」，心裡深有感觸。我自己就是處於 2.5 階段，雖然同時開 5-10 個 agent，但已經察覺到兩個瓶頸。

## 兩個瓶頸

一是注意力有限，5-6 個輪流要我審核，過度參與我會跟不上。

二是信任度不足，不敢放手叫他們全部親自跑，還是設很多人類干預的節點。

上面兩個加在一起等於惡性循環。

## 這週開始的改變

所以這週我開始嘗試改變。

第一，設置多個深夜排程，每天去掃我知識庫中逾期跟隔天要做的待辦，通通把它做掉，早上留分支待我審核。實際在背景跑的，包括我那套「夜間三件套」的維運：把健身房五題小考全面加難、答案全部改成程式驗證；把辯論社的裁判從 gemma3:4b 換成 Gemma 4，因為 gemma3:4b 自己在健身房的邏輯題都會答錯，卻負責裁定邏輯辯論、可信度存疑；順手也把電台那支 ffmpeg 卡住產不出音檔的坑修好了。這些不是空話，是真的有東西在夜裡跑。

第二，早上六點自動生成一個儀表板，上面放上待我審核的分支、非檔案修改而是對外聯絡溝通的事項，並且附上給我看的前情提要。

第三，我會叫 Agent 去讀這個儀表板，然後用 herdr 的「agent 自動操控終端機」功能，為每個需要人工審核的事項自動開 session，我再去跟這些 session 互動。

實證下來，原本要花 4-5 個小時跟 agent 協作的事情，現在降到 1-2 小時內解決。最後出去的品質也沒有下降。

下一步應該就是要進一步提高自動化跟信任度，放手讓他們各自派出 subagent，提高平行辦事的效率了。
