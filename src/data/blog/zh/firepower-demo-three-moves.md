如果你想在公司讓主管或同事**一秒對 Agentic AI 有感**，進而推動小班進階課的話，我自己最常用的三招是這幾個。

不是炫技。是針對辦公室人最痛的三個地方下手。

## 第一招：整理桌面亂檔

打開任何一個白領的電腦，你會看到桌面或下載資料夾躺著一坨東西：

- 上週開會的會議紀錄.docx
- 上週開會的會議紀錄_最終版.docx
- 上週開會的會議紀錄_最終版_確認.docx
- 上週開會的會議紀錄_最終版_確認_修改後.docx
- 從 Slack 下載的圖片 12 張
- 客戶寄來的合約 PDF 跟回信草稿
- 報名表 csv、發票 pdf、各種隨手存的螢幕截圖

火力展示的流程：

1. 開 Claude Code，把對方桌面或下載資料夾掛進來
2. 下指令：「整理這個資料夾，相同主題的檔案歸類在一起，重複的版本只留最新版（提醒我哪些被合併），給每個資料夾一個有意義的名稱」
3. Claude Code 跑完之後，原本一團亂的桌面變成 5-7 個清楚的資料夾

對方的反應通常是「天啊⋯這個我自己排兩小時都排不完」。

## 第二招：整理相互衝突的會議紀錄跟信件

這招更狠。

辦公室人最怕的是「我記得那次會議好像有講過⋯但不確定」。然後在 Slack、email、紙本筆記、Teams 之間翻半小時，最後還是不確定。

火力展示的流程：

1. 把所有跟某個議題相關的 Slack 對話、email、Teams 訊息、會議紀錄一次拖進 Claude Code
2. 下指令：「這些內容是關於 X 議題。請整理出時間線、最終決定、未解爭議、有衝突的說法（並指出衝突在哪兩個來源之間）」
3. Claude Code 產出一份 3-5 段的整理，明確標出「A 在 5/3 說可以，但 B 在 5/7 的 email 裡反對」這種衝突點

這招的殺傷力在於：**這是只有人類認真坐下來花 1-2 小時才做得出來的事**。但你的 Claude Code 5 分鐘就做完。

對方看到的不只是「快」，是「我以前以為這件事必須要花時間，現在不用了」。這個體感比任何 demo 都強。

## 第三招：迅速生成不同版本的信件

辦公室人每天最瑣碎的工作就是寫信。

同一個議題，寫給主管要正式、寫給同事要簡短、寫給客戶要客氣、寫給廠商要明確要求。同樣的內容你要重新組織 4 次。

火力展示的流程：

1. 告訴 Claude Code：「這是議題的背景（貼上 1-2 段）。我需要你幫我寫四個版本：A 給我主管，要正式並強調風險；B 給專案組同事，重點是接下來的 action item；C 給客戶，語氣客氣但要明確說明延期；D 給廠商，直接要求他們改正並給時限」
2. Claude Code 一次產出四版
3. 你只需要在每版改 1-2 句加上自己的口吻

15 分鐘的工作壓縮成 3 分鐘。

## 為什麼這三招對辦公室人特別有感

因為大部分的人電腦裡都有一個永遠混亂的資料夾。大部分的人也在花時間處理各種繁瑣文檔。**這三件事是「人人都有，但人人都討厭做」的工作**。

技術人對 AI 寫 code、debug、解 stack overflow 問題很有感。但是辦公室人不寫 code。你跟他講 hook、subagent、MCP，他眼神是空洞的。

要 demo 給辦公室人看，必須選**他們每天真的會碰到的工作**。這三招就是這樣選出來的。

## 火力展示有成果後的下一步

當你火力展示有成果之後，再跟公司談**小班進階課**。

到時候就可以調查主管的需求、參加者的需求，然後客製實作練習跟 Skill。這時候你會發現公司願意付的價錢，跟你「直接報企業培訓」時報的價錢，會有 2-3 倍的差距。

因為前者是 cold sell，後者是「我已經看見你能做什麼、我相信你」之後的價錢。

火力展示是免費的。但是**它幫你建立了信任**。後面接下來的小班、L3 顧問、長期合作，都是在這個信任之上長出來的。

如果你正在想要怎麼把 AI 工作帶進公司，先別急著推銷課程。先選一個你身邊的同事或主管，挑這三招其中一招做給他看。一週之內你就會看到他自己跑過來問「我那個 X 你可不可以幫我看一下」。

到那時候，門才真正打開。
