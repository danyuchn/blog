# Impeccable Design Context

## Design Context

### Users
讀者是對 AI 工具有興趣的中文使用者（以台灣、香港為主），通常在電腦或手機上閱讀。他們想看的不是教學文件，而是「一個真實的人怎麼用 AI 做東西、踩了什麼坑、有什麼觀點」。讀者期待的是有觀點、有溫度的文字，不是說明書。

**Author persona**: Dustin，曼谷的 GMAT 教師轉 AI 產品建造者。非工程師背景，用 Claude Code 和 Gemini 把想法實現。寫的是真實經歷，不是技術文件。

### Brand Personality
**三個詞**：個人・真實・有溫度

- 像在跟朋友講話，不是在做簡報
- 有觀點，敢說不喜歡什麼
- 不假裝自己是工程師，但認真把事情做好
- 文字密度高，但不枯燥

### Aesthetic Direction
**核心方向**：溫暖的個人出版物感，介於手寫筆記和小型獨立雜誌之間

**正面參考**：
- 個人 Substack（有個性的那種）
- 獨立書店的選書手冊美感
- 日系個人部落格（Zenn.dev、某些 note.com）

**明確反對（Anti-references）**：
- Medium：白底黑字超寬，無聊，沒個性
- Hacker News：純文字老派，沒設計
- AI cyber/neon：深色 + 紫色漸層 + 發光邊框
- 卡片堆疊 portfolio 感：每篇文章一張卡，像商業模板

**主題**：Light 和 Dark 都要精緻，不是事後補丁

### Design Principles
1. **文字第一**：排版決定一切。字型、尺寸、行距比視覺裝飾更重要
2. **個性勝過精緻**：寧可有點粗糙但有靈魂，不要光滑但無記憶點
3. **留白是設計**：不塞滿，讓內容呼吸，閱讀體驗是核心
4. **顏色要克制**：1-2 個強調色，不要彩虹，溫暖色調優先（琥珀、米白、深藍、森林綠）
5. **兩者皆精緻**：Light mode 有質感，Dark mode 是「昏黃燈光下的咖啡館」，不是「駭客終端機」
